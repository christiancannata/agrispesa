Installation

First download the plugin file zip file, then there are two ways for installing:
 
1) The best way to install is via the Plugins > Add New Plugins section in the Wordpress Backend.
 
2) You can also manually bu unzipping plugin file in /wp-content/plugins directory
 
Once the plugin is installed please activate.
It will add a new menu voice WooCommerce Checkout Fields menu by which you can add the fields and the visibility conditions by which the additional fields will be showed on checkout page.

Configuration

By accessing to the WooCommerce Checkout Fields menu you can add the new billing/shipping fields that will be lately displayed during the checkout process after the default WooCommerce form fields. Each field can be have its own visibility conditions. Fields will be showed on checkout page only if the configured condition (if any) will be satisfied.

Field types

You can create the following field types:

  1. Text
  2. Text area
  3. Number
  4. Telephone
  5. Email
  6. Password
  7. Select/Multiselect
  8. Date
  9. Time
  10. Country
  11. State

 

For each field you can configure its own option like CSS classes to assign to the input field, placeholder text, show on emails, show on order details page, etc.

 

Select/Multiselect field note

To secify values and labels use the following format:

value1 : value 1
value2 : value 2

In which each line has a value and label separated by the : character.

Note for WPML users: you can use different labels for each language but you must use same values.
For example: " value_1 : Green" for English and "value_1 : Verde" for Italian.

State field note

Can be selected only the states/provinces for the allowed sell countries configured in the WooCommerce -> Settings -> General menu.

WPML compability note

If WPML is detected, the plugin will automatically show multilanguage input fields for labeles, placehonders, etc. In this way you can enter different texts for each language.

Field visibility condition

For each field can be bounded several and/or conditional rules that will make the field visible only if satisfied. A condition can be have 4 typologies:

  1. Product
  2. Category
  3. Cart
  4. User

 

Product/Category

Allows you to make the field visible according the followin options:

  1. Cart/Stock quantity
  2. Stock status
  3. Lenght
  4. Height
  5. Width
  6. Weight
  7. Volume
In case of multiple product selections, selected values can be considered for each item or as sum of items values. You can also optionally configure the rule in order to be satisfied if all the selected product are in cart or to apply just only to the product that actually are in cart.

Cart

Allows you to make the field visible according the followin options:

  1. Sub total, Total, Shippings or Discount (with or without taxes applied)
  2. Subtotal, Total, Shipping or Discount taxes
  3. Sum of item quantities, weights or volumes
  4. Number of distinct item on cart

User

Field will be visible if the current user has or hasn't one of the selected roles

Frontend integration

Additional fields will be seamlessly integrated into the Billing/Shipping form if the conditional rules will be satisfied

 

And optionally reported into the user oder details page and notification emails
